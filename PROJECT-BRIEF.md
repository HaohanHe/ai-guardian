### 遇到的问题 ❌

**核心问题：AutoClaw 的 Skill 机制不能阻止命令执行**

经过调研发现：
- **OpenClaw** (原版): `before_tool_call` 可以 **阻止执行** ✅
- **OpenCode** (AutoClaw 基于它): `tool.execute.before` **不能阻止** ❌

**这就是为什么我们的 Skill 无法拦截删除命令！**

### 技术路线调整

既然 OpenCode/AutoClaw 的 hook 不能阻止执行，我们需要：

#### 方案 A：系统级拦截（推荐）
- **Windows**: API Hooking (Detours、EasyHook)
- **Linux**: LD_PRELOAD
- **Mac**: DYLD_INSERT_LIBRARIES
- **效果**: 在操作系统层面拦截所有进程创建

#### 方案 B：终端代理
- 替换系统 shell (PowerShell/CMD/Bash)
- 所有命令经过 Guardian 检查后再执行
- **效果**: 100% 拦截率，但用户体验稍差

#### 方案 C：修改 AutoClaw 源码
- 如果 AutoClaw 开源，直接改源码添加拦截逻辑
- **效果**: 最彻底，但依赖 AutoClaw 配合

#### 方案 D：与 AutoClaw 官方合作
- 联系 AutoClaw 团队，争取官方支持
- 成为 AutoClaw 官方推荐的安全插件
- **效果**: 最佳，但需要商务谈判

### 任务 1：解决 Skill 加载问题（优先级：🔴 最高）

**负责人**：后端开发工程师

**目标**：让 AutoClaw 正确加载 AI Guardian Skill，拦截危险命令

**具体工作：**
1. 研究 AutoClaw 的 Skill 加载机制
   - 查看 AutoClaw 日志：`C:\Users\LENOVO\.openclaw-autoclaw\logs\gateway.log`
   - 确认 Skill 是否被识别
   
2. 尝试编译 Skill
   ```bash
   cd skills/autoclaw/ai-guardian-skill
   npm install
   npx tsc index.ts --target ES2020 --module ESNext --outDir dist
   ```
   
3. 测试不同的 Hook 注册方式
   - 尝试 `ctx.on("before_tool_call", ...)`
   - 尝试 `api.on("before_tool_call", ...)`
   - 尝试 `registerHook("before_tool_call", ...)`

4. 如果 Skill 不行，尝试其他方案
   - 修改 AutoClaw 源码（如果有权限）
   - 使用系统级 Hook（LD_PRELOAD、API Hooking）
   - 使用终端代理（替换 PowerShell/CMD）
